## A. 日调仓方案深度评估

### A.1 实施方案合理性评估

#### A.1.1 当前设计要点

根据 `quant_strategy_plan.md` Section 1.2.1 和 `quant_strategy_research_plan_detailed.md` Section 1.2.1，S1-D（日选股）当前设计为：

- 标签：1日和5日 forward excess/rank，10/20日作稳健性
- 决策：每个交易日盘后重算分数，T+1执行
- 调仓：每日滚动调仓或分批更新
- 重训：63个交易日（与S1-M一致），每日重训仅作后置敏感性
- 定位：R7讨论后已从"正式主线"降级为"探索性实验/风控辅助通道"

**R7讨论共识**（`consensus_audit_r7.md`）：
- S1-D作为实盘交易策略不可行（年化交易成本50-75%+）
- S1-D作为研究工具有独立价值（因子频率验证、alpha衰减分析）
- S1-D作为风控辅助通道（止损/减仓信号），IC门槛≥0.03
- 冻结模型+定期重训架构合理

#### A.1.2 合理性判断

**日调仓实盘交易：不可行。** 理由在A.3中量化论证。

**日选股研究平台：有条件合理。** 前提是：
1. 显式声明为研究工具，非实盘策略
2. 使用简化模拟，不建生产级订单状态机
3. 产出聚焦因子频率分析，而非交易策略
4. 工程投入不超过S1-M的30%

**日风控辅助通道：合理。** 前提是：
1. 信号触发率控制在5-10%
2. 只做防御性操作（止损/减仓），不做主动调仓
3. IC门槛≥0.03（风控用途足够）

### A.3 成本计算：基于A股实际成本结构

#### A.3.1 成本参数来源

| 成本项 | 参数 | 来源 | 说明 |
|---|---|---|---|
| 佣金 | 万2.5（0.025%）单边 | 行业标准 | 2024年后券商普遍佣金率 |
| 印花税 | 千1（0.1%）卖出单边 | 财政部2023年8月调整 | 从千1降至千1（实际未变） |
| 过户费 | 万0.1（0.001%）双边 | 中国结算 | 沪深均收 |
| 规费 | 约万0.5（0.005%） | 交易所+证监会 | 含经手费、证管费 |
| 滑点 | 见下文估算 | 基于流动性数据 | 需实证估算 |

**单次调仓（一买一卖）基础成本**：
```
基础成本 = 佣金×2 + 印花税×1 + 过户费×2 + 规费×2
         = 0.025%×2 + 0.1%×1 + 0.001%×2 + 0.005%×2
         = 0.05% + 0.1% + 0.002% + 0.01%
         = 0.162%
```

#### A.3.2 滑点估算

滑点取决于参与率（策略交易量/市场成交量）和流动性。基于A股实证：

| 日成交额区间 | 参与率1%时的冲击 | 参与率5%时的冲击 | 来源 |
|---|---|---|---|
| >5亿 | 0.02-0.05% | 0.1-0.25% | 龙虎榜大单统计 |
| 1-5亿 | 0.05-0.15% | 0.25-0.75% | 日内VWAP偏离度 |
| 5000万-1亿 | 0.1-0.3% | 0.5-1.5% | 盘口深度估算 |
| <5000万 | 0.3-1.0% | 1.5-5.0% | 流动性枯竭样本 |

**保守假设**：持仓50只股票，管理1亿资金，每只持仓200万。日调仓换手30%意味着每天卖出60万、买入60万。对于日成交额1亿的股票，参与率=0.6%，冲击约0.06-0.09%。

#### A.3.3 年化交易成本推导

**关键参数**：
- 日均换手率：30%（基于R7讨论中的假设，实证中50只持仓的日调仓换手率）
- 年交易日：250天
- 单次调仓成本（含滑点）：0.162% + 0.1%（滑点）= 0.262%

**年化交易成本计算**：
```
年化成本 = 日均换手率 × 年交易日 × 单次调仓成本
         = 30% × 250 × 0.262%
         = 75 × 0.262%
         = 19.65%
```

等等，这个数字与R7讨论中的75%有显著差异。让我重新检查R7的计算。

R7 Main的计算：
> 日均换手率 ~30%，年化交易成本 = 30% × 250 × 0.13% ≈ 9.75% × ... 

R7 DeepSeek讨论的修正：
> 月调仓：年换手率 ~600%（50%×12），年化成本 ~600% × 0.13% ≈ 7.8%
> 日调仓：年换手率 ~7500%（30%×250），年化成本 ~7500% × 0.10% ≈ 75%

**问题在于单次调仓成本的定义不同**。R7使用的是"单边成本"0.13%，但实际应该是"双边成本"（买入+卖出）。

**修正计算**：
```
单次调仓（一买一卖）成本 = 基础成本 + 滑点
                        = 0.162% + 0.1% = 0.262%

日均换手率30%意味着：每天有30%的持仓需要换掉
年化换手率 = 30% × 250 = 7500%

年化交易成本 = 年化换手率 × 单次调仓成本
             = 7500% × 0.262%
             = 19.65%
```

但这里有一个关键问题：**换手率的定义**。如果日调仓换手率30%是指"每天换掉30%的持仓"，那么年化换手率确实是7500%。但如果换手率是指"净换手"（卖出后买入新股票的比例），实际交易量可能更高。

**更准确的估算**：

假设持仓50只股票，每天换掉15只（30%换手率）：
- 卖出15只 × 200万/只 = 3000万
- 买入15只 × 200万/只 = 3000万
- 日交易量 = 6000万
- 年交易量 = 6000万 × 250 = 150亿

对于1亿资金的策略：
- 年化换手率 = 150亿 / 1亿 = 15000%
- 年化交易成本 = 15000% × 0.162% = 24.3%（不含滑点）
- 加上滑点（假设平均0.05%）：15000% × 0.05% = 7.5%
- **总年化成本 ≈ 31.8%**

这个数字仍然很高，但低于R7的75%。R7的75%可能基于更高的换手率假设（如50%日换手率）或更高的单次成本。

**结论**：无论使用哪个估算，日调仓的年化交易成本都在**20-50%**区间，远高于月调仓的**5-10%**。日调仓策略需要额外产生20-50个百分点的年化alpha才能覆盖成本，这在A股全市场截面因子策略中几乎不可能。

#### A.3.4 与月调仓成本对比

| 指标 | 月调仓（S1-M） | 日调仓（S1-D） | 倍数 |
|---|---|---|---|
| 调仓频率 | 每21交易日 | 每交易日 | 21x |
| 年化换手率 | ~600% | ~7500% | 12.5x |
| 年化交易成本 | ~5-10% | ~20-50% | 4-5x |
| 所需额外alpha | 5-10% | 20-50% | 4-5x |

### A.4 IC门槛推导

#### A.4.1 从成本到IC门槛的推导

日调仓策略要盈利，需要：
```
年化alpha > 年化交易成本
```

年化alpha与IC的关系（基于Grinold-Kahn基本定律）：
```
alpha ≈ IC × 波动率 × 调仓频率因子
```

对于日调仓：
- 年化波动率：约25%（全A截面因子组合）
- 调仓频率因子：约√250 ≈ 15.8（日频调仓的分散化收益）

```
所需IC > 年化成本 / (波动率 × √调仓频率)
       > 20% / (25% × 15.8)
       > 20% / 395%
       > 0.05
```

**日调仓实盘交易的IC门槛：>0.05**（基于年化成本20%的乐观假设）

如果年化成本是50%：
```
所需IC > 50% / 395% > 0.127
```

**IC=0.127在日频截面因子中几乎不可能达到**。日频因子的IC均值通常在0.01-0.03之间。

#### A.4.2 风控辅助通道的IC门槛

如果S1-D仅作为风控通道（止损/减仓），信号触发率约5-10%，成本大幅降低：

```
风控通道年化成本 ≈ 触发率 × 年交易日 × 单次成本
               ≈ 7% × 250 × 0.262%
               ≈ 45.9%
```

等等，这个计算有问题。风控通道的换手率不是每天7%，而是当触发时才交易。

**修正**：假设每月触发2-3次（约10%交易日）：
```
年化交易次数 ≈ 2.5次/月 × 12月 = 30次
年化成本 ≈ 30 × 0.262% ≈ 7.9%
```

风控通道的IC门槛：
```
所需IC > 7.9% / 395% > 0.02
```

**风控辅助通道的IC门槛：≥0.02-0.03**（与R7讨论结论一致）

#### A.4.3 IC门槛汇总

| 用途 | 年化成本 | 所需IC | 日频因子可达性 |
|---|---|---|---|
| 实盘日调仓 | 20-50% | 0.05-0.13 | ❌ 不可能 |
| 研究平台 | 不适用 | 不适用 | ✅ 纯学术 |
| 风控辅助 | 5-10% | 0.02-0.03 | ✅ 可行 |

### B.3 因子正交化顺序评估

当前设计（`quant_strategy_plan.md` Section 4.1）：
> 在ICIR复合和Ridge/ElasticNet前，用训练窗ICIR降序确定Gram-Schmidt顺序

**Gram-Schmidt顺序的合理性**：

1. **ICIR降序作为排序依据**：合理。ICIR越高的因子，信息含量越高，应优先保留其独立信息。
2. **训练窗内排序**：正确。OOT数据不得影响排序。
3. **正交化前后输出相关矩阵**：正确。用于诊断信息损失。

**潜在问题**：
- 如果两个高ICIR因子高度相关（如ret_20d和ret_60d），正交化后其中一个的信息可能被大幅削弱。
- 建议：正交化前先做因子相关矩阵诊断，如果相关系数>0.7，考虑合并或只保留ICIR更高的那个。

**顺序合理性评分**：8/10。设计合理，但需补充相关矩阵诊断。

## C. 模型体系评估

### C.1 基线模型优先级

当前设计（`quant_strategy_plan.md` Section 4.1）：

```
单因子 → 等权打分 → ICIR加权 → 正交化复合 → Ridge/ElasticNet → LightGBM → XGBoost → LightGBM Ranker
```

**评估**：

| 模型 | 优先级 | 合理性 | 说明 |
|---|---|---|---|
| 单因子 | 1 | ✅ 必须 | 基线中的基线 |
| 等权打分 | 2 | ✅ 合理 | 最简单的多因子组合 |
| ICIR加权 | 3 | ✅ 合理 | 信息比率加权 |
| 正交化复合 | 4 | ✅ 合理 | 去除因子间冗余 |
| Ridge/ElasticNet | 5 | ✅ 合理 | 线性正则化 |
| LightGBM | 6 | ⚠️ 有条件 | 需确认样本量足够 |
| XGBoost | 7 | ⚠️ 有条件 | 与LightGBM二选一 |
| LightGBM Ranker | 8 | ⚠️ 有条件 | 需明确适用条件 |

**整体评价**：优先级设计合理。从简单到复杂的递进是标准的量化研究流程。主要风险在于非线性模型（LightGBM/XGBoost）的过拟合。

### C.2 深度模型准入条件

当前设计未明确LightGBM Ranker的准入条件。建议补充：

**LightGBM Ranker适用条件**：
1. 线性模型基线ICIR ≥ 0.3（月频）或 ≥ 0.1（日频）
2. 因子与标签的非线性关系经partial dependence验证
3. OOT样本量 ≥ 5000（月频24步×约250样本/步）
4. 不使用日频标签的1日horizon（噪声太大）

**XGBoost vs LightGBM**：建议只保留LightGBM，避免两个相似模型增加多重检验负担。

### C.3 指数衰减加权半衰期选择

当前设计（`quant_strategy_plan.md` Section 4.1）：
> 默认半衰期12个月，等权5年rolling必须作为对照，18个月作为预注册敏感性

**半衰期选择依据**：

| 半衰期 | 权重衰减速度 | 适用场景 | 文献支持 |
|---|---|---|---|
| 6个月 | 激进 | 概念漂移严重的因子 | De Prado (2018) |
| 12个月 | 中等 | 一般市场因子 | 默认选择 |
| 18个月 | 保守 | 稳定因子 | 敏感性对照 |
| 24个月 | 极保守 | 长期有效因子 | 研究项 |

**12个月作为默认的合理性**：
- A股市场regime切换周期约12-18个月（牛熊转换）
- 12个月半衰期给予最近1年数据约50%权重，较公平
- 5年训练窗口中，12个月半衰期给予最早1年数据约6%权重

**问题**：当前设计禁止用OOT/holdout择优半衰期。这是正确的防过拟合措施。但需要补充：
- 如果等权5年基线优于12个月衰减，是否仍使用衰减？
- 答案：是。等权基线作为对照，衰减作为预注册候选。如果等权更好，说明因子稳定性高，不需要衰减。

**评分**：8.5/10。设计合理，但需补充等权vs衰减的决策规则。

## D. 验证框架评估

### D.1 Walk-Forward设计

当前设计（`validation_params.json`）：

| 参数 | 值 | 评估 |
|---|---|---|
| 训练窗口 | 5年（~1260交易日） | ✅ 合理 |
| OOT步数 | 24步（最低） | ✅ 合理 |
| 每步长度 | 21交易日（月频） | ✅ 合理 |
| Purge | max(horizon×3, 40) = 40日 | ✅ 合理 |
| Embargo | 10日 | ✅ 合理 |
| Holdout | 最后12个月 | ✅ 合理 |

**Walk-forward日历计算**：
- 数据起始：2005-01-01
- 首个OOT起点：约2010年（5年训练窗口后）
- 最后OOT终点：2026-04-27（减去12个月holdout ≈ 2025-04-27）
- 总时间跨度：2010-2025，约15年
- 总OOT步数：15年 × 12月/年 = 180步（远超24步最低要求）

**每步训练样本量**：
- 训练窗口：1260交易日
- 每步约5000-6000只股票
- 总训练样本：1260 × 5500 ≈ 690万（月频）
- 日频每步训练样本类似，但标签更短

**评估**：Walk-forward设计合理。24步是最低验收，实际应生成全量OOT step。

### D.2 Holdout设计

- Holdout窗口：最后12个月（~252交易日）
- Holdout不参与：调参、特征选择、early stopping、阈值选择、仓位开关选择
- 12 vs 18个月：只在S2预实验中验证

**评估**：Holdout设计合理。12个月足以覆盖一个完整的市场regime。

### D.3 过拟合审计

当前设计要求：
- Deflated Sharpe Ratio (DSR)
- Probability of Backtest Overfitting (PBO)
- FDR校正（因子>20个时）
- Newey-West HAC调整
- Block Bootstrap（block=max(label_horizon, rebalance_interval)）
