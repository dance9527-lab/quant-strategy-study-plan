## 分歧点 1：统计功效与 MDE（Minimum Detectable Effect）

### 我的立场：MDE 是 P0，但 R1 的定量推理有误，需修正

**R1 提出的问题**：我在 R1 中将"统计功效分析缺失"列为 P0，认为 24 步 OOT 可能无法检测到经济上有意义的 alpha。其他 agent 均未明确提出此问题。

**为什么 MDE 是 P0（定性论证）**：

1. **24 步是 keep 决策的最低门槛**：如果 24 步的统计功效不足，那么基于 24 步 OOT 的 keep/reject 决策将不可靠——要么错过真 alpha（假阴性），要么接受假 alpha（假阳性）。
2. **功效分析是实验设计的前置条件**：在 frequentist 框架中，样本量（此处为 OOT 步数）和检测功效是实验设计的第一步。不做功效分析就启动实验，等于不知道实验需要多长时间才能得出结论。
3. **holdout 窗口同样面临功效问题**：12 个月 holdout（约 12 个调仓周期）检测年化 Sharpe=0.3 的 alpha 的功效可能只有 40-60%，这意味着即使 S1 通过了 OOT 验证，holdout 仍有相当概率无法确认。

**具体计算建议**：

对于月频 IC 序列（24 步 OOT），MDE 的计算框架如下：

**单样本 t 检验功效分析**：
- 零假设 H0：IC 均值 = 0
- 备择假设 H1：IC 均值 = δ（真实 alpha）
- 显著性水平 α = 0.05（单侧）
- 样本量 n = 24
- IC 标准差 σ（需从历史数据估计）

功效公式：`power = Φ(δ√n/σ - z_{1-α})`

反解 MDE：`MDE = σ(z_{1-α} + z_{power}) / √n`

| IC 标准差 σ | MDE (power=80%) | MDE (power=90%) | 对应年化 Sharpe* |
|---|---|---|---|
| 0.04 | 0.022 | 0.027 | 0.55 / 0.68 |
| 0.06 | 0.033 | 0.041 | 0.83 / 1.03 |
| 0.08 | 0.044 | 0.054 | 1.10 / 1.35 |

*年化 Sharpe 近似 = IC_mean / IC_std × √12

**关键洞察**：
- 如果 IC 标准差 = 0.06（A 股月频因子的典型水平），24 步 OOT 在 α=0.05、power=80% 下只能检测到 IC 均值 ≥ 0.033 的 alpha，对应年化 Sharpe ≈ 0.83。
- 这意味着年化 Sharpe < 0.83 的弱 alpha（这在 A 股市场中非常常见）在 24 步 OOT 中**没有足够的统计功效被检测到**。
- 如果目标是检测年化 Sharpe = 0.3（更现实的 alpha 水平），对应 IC 均值 ≈ 0.012，需要的样本量为 `n = (σ(z_{0.95} + z_{0.80}) / δ)² = (0.06 × 2.485 / 0.012)² ≈ 154` 步——远超 24 步。

**⚠️ R1 推理修正**：

我在 R1 中声称"IC 自相关导致实际误报率偏高"，这个推理需要修正。通过 R2 的定量计算（见分歧点 6），在零假设（IC 均值=0）下，正自相关实际上**降低**了 5/6 门槛的误报率（从独立假设的 10.9% 降至 ρ=0.3 时的约 2.5%）。R1 的推理错误在于混淆了"零假设下的误报率"和"备择假设下的漏报率"。

**最终建议**：
1. **在 Phase A0.1 中增加功效分析任务**：基于仓库中已有的因子 IC 历史数据，估计 σ_IC，计算 24 步 OOT 和 12 个月 holdout 的 MDE。
2. **在 validation_params.json 中预注册 MDE**：明确"本实验的最小可检测 IC 均值为 X，对应年化 Sharpe = Y"。
3. **如果 MDE 过大，调整实验设计**：可能的调整包括增加 OOT 步数（从 24 提高到 36 或更多）、使用 one-sided test（已采用）、或放宽 α 水平（如从 0.05 到 0.10）。

## 分歧点 3：CSRP 命中窗口

### 我的立场：支持 Review Agent 的 P0 判定，命中窗口必须预注册

**Review Agent 的问题**：CSRP 误报率框架要求"命中窗口"，但文档未定义"命中"的具体标准——信号触发后 5 日、10 日、20 日都可以算"命中"，这会导致误报率估计的自由度过高。

**我的分析**：

1. **命中窗口应与标签持有期一致**：S1-M 的标签是 20 日（月频），因此 CSRP 的主口径命中窗口应为**信号后 1 个 OOT step（约 20 个交易日）**。这是因为在 20 日标签框架下，"命中"的自然定义是"信号发出后的下一个 OOT step 的成熟 IC < 0"。

2. **敏感性分析应覆盖多窗口**：除主口径外，应报告信号后 2 步（40 日）和 3 步（60 日）的命中率，以检验信号的持续性。

3. **命中标准应为组合收益而非 IC**：CSRP 信号的最终目的是预警组合表现恶化，因此"命中"应定义为"信号发出后 N 步内，组合收益显著低于 benchmark"，而非仅看 IC。IC 为负但组合收益为正（可能是成本改善）不应算作命中。

4. **最低信号数门槛**：如 DeepSeek R1 和 Review R1 均指出，`n_signals < 5` 时统计功效极低。建议：`n_signals < 5` 时误报率统计仅作参考，不得作为 tighten-only 依据。

**最终建议**：
- 主口径命中窗口：信号后 1 个 OOT step（20 个交易日）
- 敏感性：信号后 2 步（40 日）和 3 步（60 日）
- 命中标准：组合收益 < benchmark 收益（扣除成本后），而非仅 IC < 0
- 最低信号数：`n_signals >= 5` 才可作为 tighten-only 依据
- 在 validation_params.json 中预注册以上所有参数

## 分歧点 4：分层裁剪与 Capital Overlay 执行顺序

### 我的立场：支持 Review Agent 的 P0 判定，执行顺序必须明确

**Review Agent 的问题**：分层裁剪（总换手 → 行业 → 个股）和 capital overlay（乘以 capital_multiplier）的执行顺序未明确。如果顺序错误，会导致权重逻辑矛盾。

**正确执行顺序**：

```
1. 选股模型输出原始目标权重 w_raw（总和 = 1.0）
2. 分层裁剪：
   a. 按总换手上限等比例缩减 → w_1（归一化到 1.0）
   b. 按行业上限对超限行业等比例缩减 → w_2（归一化到 1.0）
   c. 按个股上限裁剪单票变化 → w_3（归一化到 1.0）
3. Capital overlay：
   a. 查找当前市场状态 → capital_multiplier
   b. 目标权重 = w_3 × capital_multiplier
   c. 现金权重 = 1 - capital_multiplier
4. 输出：目标权重 + 现金权重 + 裁剪审计日志
```

**为什么这个顺序是正确的**：

1. **裁剪在 overlay 之前**：裁剪的目的是确保组合的换手、行业和个股集中度在可控范围内。如果先做 overlay（乘以 <1 的 multiplier），再做裁剪，那么裁剪的约束是在缩小后的权重上计算的，会导致实际约束变松。

2. **归一化在裁剪层之间**：每层裁剪后归一化到 1.0，确保后续层的约束计算基于正确的总权重。如果不归一化，行业上限的计算基准会偏小。

3. **Overlay 是最后一步**：capital_multiplier 是资金层面的调整，不应影响选股层面的换手和集中度约束。现金部分不参与裁剪。

4. **与 T+1 执行约束的兼容**：减仓受 T+1 约束。如果 capital_multiplier 从 1.0 降到 0.5（牛市→熊市），T 日只能卖出到 T+1 的 target。文档应明确"减仓按 T+1 open 执行，不支持 T+0 卖出"。

**对 Review Agent 提出的"裁剪后权重归一化"问题的回应**：

裁剪后归一化到 1.0 是正确的——归一化后的 1.0 代表"满仓"（不考虑 capital overlay）。capital overlay 的作用正是将满仓权重缩放到目标仓位。如果裁剪后不归一化，capital overlay 的 multiplier 会同时承担"裁剪导致的权重缩减"和"市场状态导致的仓位调整"两个角色，逻辑不清晰。

**最终建议**：
1. 在执行规范 §7.1 中明确以上执行顺序
2. 明确"裁剪在 capital overlay 之前，裁剪后归一化到 1.0"
3. 明确"减仓受 T+1 约束，T 日判定状态变化，T+1 执行减仓"
4. 输出裁剪审计日志：裁剪前权重、裁剪后权重、超限资产/行业、capital_multiplier、最终目标权重

## 分歧点 5：Sleeve FDR 计入规则

### 我的立场：Sleeve 必须计入 FDR 实验族，影响显著

**R1 提出的问题**：我在 R1 中指出"sleeve 是否计入同一实验族"未明确，且 FDR 与 sleeve 的交互需要定义。

**详细分析**：

1. **Sleeve 是否应计入 FDR 实验族？——是的**

   每个 sleeve 本质上是一次独立的"因子 × 选股规则"组合尝试。如果 `baseline_top_score` 通过了 OOT 验证，但 `upper_middle_score_p80_p95` 没有通过，而我们只报告前者，这就是**选择性报告**——正是 FDR 要校正的问题。

   从实验族定义来看，sleeve 的差异在于"选股规则"（选哪些股票），这与"因子 × 标签 × 模型 × 半衰期 × 训练窗口 × 正交化分支 × 执行规则"中的"执行规则"维度是同一类变异。

2. **对实验总数的影响有多大？**

   当前文档定义的实验族维度：
   - 因子：预注册约 10-15 个
   - 标签：3 种（20日、1日风险、1日执行）
   - 模型：7 种（单因子、等权、ICIR、Gram-Schmidt、Ridge、LightGBM、Ranker）
   - 半衰期：2 种（12月、18月）
   - 训练窗口：1 种（5年）
   - 正交化分支：2 种（有/无）
   - 执行规则：2 种（月度、日度）

   不含 sleeve 的实验族大小 ≈ 15 × 3 × 7 × 2 × 1 × 2 × 2 = 2,520 次尝试

   加入 5 个预注册 sleeve 后：
   - 如果每个 sleeve 独立计数：2,520 × 5 = 12,600 次尝试
   - 如果 sleeve 只对 S1-M 适用（约 1/3 的实验）：2,520 × 1/3 × 5 + 2,520 × 2/3 = 4,200 + 1,680 = 5,880 次尝试

   **影响评估**：从 2,520 到 5,880-12,600，实验总数增加 2.3x-5x。BH-FDR 校正的 p 值阈值会相应变严。例如：
   - 2,520 次尝试中，第 100 个最小 p 值的 BH 阈值 ≈ 100/2,520 × 0.05 = 0.00198
   - 5,880 次尝试中，同样的 BH 阈值 ≈ 100/5,880 × 0.05 = 0.000850

   这意味着 sleeve 的引入会使 FDR 校正严格约 2.3 倍。

3. **建议的 sleeve FDR 规则**：

   **方案 A（推荐）：Sleeve 共享 test_family_id，但独立计入 attempt_count**
   - 所有 5 个 sleeve × 同一因子 × 同一模型 × 同一标签的组合属于同一 test_family
   - 每个 sleeve 的 keep 决策必须通过包含所有 sleeve 尝试的 FDR 校正
   - 优点：防止 sleeve 间的选择性报告，同时允许 sleeve 内的独立评估

   **方案 B：Sleeve 独立成族**
   - 每个 sleeve 形成独立的 test_family
   - 优点：sleeve 间不互相惩罚
   - 缺点：如果测试了 5 个 sleeve 只报告最好的一个，无法被 FDR 捕获

   **方案 A 更符合 FDR 的设计初衷**——防止在同一实验族内的选择性报告。

**最终建议**：
1. 采用方案 A：sleeve 共享 test_family_id，独立计入 attempt_count
2. 在 validation_params.json 中预注册 sleeve 清单和 test_family 归属规则
3. 在 Phase A0.1 中完成 sleeve 的 FDR 影响评估（基于预注册的因子和模型数量）
4. 如果 FDR 影响过大（如 BH 阈值 < 0.001），考虑减少预注册 sleeve 数量（从 5 个减到 3 个核心 sleeve）

## 分歧点 6：Concept Shift 独立性假设——定量修正

### 我的立场：R1 的定性判断方向正确但定量推理有误，需修正

**R1 的原始判断**：我在 R1 中声称"IC 自相关导致实际误报率偏高"。通过 R2 的定量分析，这个判断需要修正。

**定量分析框架**：

**模型设定**：
- 零假设 H0：IC 均值 = 0（无 alpha）
- IC 步间相关结构：AR(1)，lag-1 自相关系数 = ρ
- 6 步 OOT IC 序列：(X₁, X₂, ..., X₆) ~ N(0, Σ)
- Σ 为等相关矩阵，对角线 = 1，非对角线 = ρ

**关键概率计算**：

对于等相关的多元正态分布，可以使用如下表示：
```
X_i = √ρ · Z₀ + √(1-ρ) · Z_i
```
其中 Z₀, Z₁, ..., Z₆ 是独立的标准正态变量。

**P(X_i < 0) = 0.5**（无论 ρ 是多少，因为分布关于 0 对称）

**P(所有 6 步 IC < 0)**：

令 a = √(ρ/(1-ρ))，则：
```
P(all 6 < 0) = E[Φ(-a·Z₀)⁶] = ∫ Φ(-a·z)⁶ · φ(z) dz
```

通过数值积分（梯形法，步长 0.1，范围 [-6, 6]）：

| ρ | a = √(ρ/(1-ρ)) | P(all 6 < 0) |
|---|---|---|
| 0.0 | 0.000 | 0.01563 |
| 0.3 | 0.655 | 0.0129 |
| 0.5 | 1.000 | 0.0108 |
| 0.7 | 1.528 | 0.0085 |

**P(恰好 5 步 IC < 0)**：

对于等相关的多元正态分布，所有 6 种"恰好 5 负 1 正"的模式概率相等。以 (neg,neg,neg,neg,neg,pos) 为例：
```
P = E[Φ(-a·Z₀)⁵ · Φ(a·Z₀)]
```

通过数值积分：

| ρ | P(恰好 5 < 0) [单个模式] | P(恰好 5 < 0) [6 种模式合计] |
|---|---|---|
| 0.0 | 0.01563 | 0.09375 |
| 0.3 | 0.0021 | 0.0124 |
| 0.5 | 0.0012 | 0.0073 |
| 0.7 | 0.0005 | 0.0029 |

**P(5 或更多步 IC < 0) = P(恰好 5) + P(全部 6)**：

| ρ | P(≥5 < 0) | 相对独立假设的变化 |
|---|---|---|
| 0.0 (独立) | **10.94%** | 基准 |
| 0.3 | **2.53%** | ↓ 77% |
| 0.5 | **1.81%** | ↓ 83% |
| 0.7 | **1.14%** | ↓ 90% |

**关键发现**：

**在零假设（IC 均值 = 0）下，正自相关实际上降低了 5/6 门槛的误报率，而非增加。** 这与我 R1 的判断相反。

**直觉解释**：
- 正自相关使相邻步的 IC 更可能同号
- 在零假设下，这意味着要么全部 6 步都是负的（概率随 ρ 增加而增加），要么全部 6 步都是正的（概率也增加）
- "恰好 5 负 1 正"的混合模式概率随 ρ 增加而**急剧下降**
- 由于 P(恰好 5) 的下降速度快于 P(全部 6) 的增长速度，总概率 P(≥5) 随 ρ 增加而下降

**R1 推理错误的原因**：

我在 R1 中混淆了两个不同的问题：
1. **零假设下的误报率**（FPR）：正自相关 → FPR 降低（本分析）
2. **有效样本量的缩减**：正自相关 → 有效独立样本数减少 → 检测真 alpha 的功效降低

R1 将"有效样本量缩减"错误地等同于"误报率升高"。实际上，有效样本量缩减主要影响的是**功效**（检测真 alpha 的能力），而非**误报率**（在无 alpha 时错误报警的概率）。

**对 10.9% 的修正**：

| 场景 | 误报率 |
|---|---|
| 独立假设（文档当前值） | 10.9% |
| ρ = 0.3（保守估计） | 2.5% |
| ρ = 0.5（中等估计） | 1.8% |
| ρ = 0.7（激进估计） | 1.1% |

**结论**：文档中的 10.9% 是**高估**了实际误报率（在零假设下）。如果 IC 步间存在正自相关，实际误报率会低于 10.9%。这意味着 5/6 门槛比文档认为的更保守。

**但这不意味着没有问题**：

虽然误报率降低了，但**功效**（检测真 alpha 的能力）也降低了。如果 IC 均值为负（策略确实在恶化），正自相关使连续 6 步 IC 为负的概率增加（从独立假设的 1.56% 增加到 ρ=0.3 时的 1.29%——等等，这里概率反而降低了？让我重新检查）。

实际上，P(all 6 < 0) 在 ρ=0.3 时为 0.0129，低于独立假设的 0.0156。这同样是因为正自相关使分布更集中，极端事件（全部为负）的概率降低。

**真正的风险不是误报率，而是功效**：如果策略确实在恶化（IC 均值为负），正自相关使连续步更可能同号，因此检测到恶化信号的概率应该增加。但零假设下的计算显示概率反而降低——这是因为零假设设定 IC 均值 = 0，而非 IC 均值为负。

**备择假设下的功效分析**（IC 均值 = -0.02，即策略恶化）：

在备择假设下，P(IC < 0) > 0.5。令 p = P(IC < 0 | μ = -0.02, σ = 0.06) = Φ(0.02/0.06) = Φ(0.333) ≈ 0.63。

独立假设下：P(≥5 < 0) = C(6,5) × 0.63⁵ × 0.37 + 0.63⁶ = 6 × 0.0992 × 0.37 + 0.0625 = 0.2204 + 0.0625 = 0.283

ρ = 0.3 时（需要复杂的多元正态计算，此处简化估计）：由于正自相关使连续步更可能同号，P(≥5 < 0) 应高于独立假设，可能在 0.35-0.40 范围。

**这意味着**：正自相关在备择假设下**提高了**检测功效（当策略确实在恶化时，更容易触发 yellow 信号），这与零假设下的误报率降低是一致的。

**最终建议**：
1. **修正文档中的误报率说明**：将"5/6 门槛的误报率为 10.9%"修正为"在独立假设下为 10.9%；在正自相关下实际误报率更低（ρ=0.3 时约 2.5%），但检测功效也会变化"
2. **通过 block bootstrap 估计实际误报率和功效**：使用实际 OOT IC 序列，通过 block bootstrap 保留自相关结构，估计 P(≥5 < 0 | H0) 和 P(≥5 < 0 | H1)
3. **关注功效而非误报率**：5/6 门槛的真正风险是功效不足（策略恶化时检测不及时），而非误报率过高
4. **R1 的定性建议仍然有效**：文档应注明 10.9% 的独立性假设，并通过实际数据验证
