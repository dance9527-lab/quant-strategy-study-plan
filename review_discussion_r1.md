# Review Agent 讨论回应 - Round 1

## 分歧1：因子库策略

**我的立场：支持 Main 的"先简单后复杂"，但要求明确写入另类数据扩展时间线。**

传统因子 alpha 衰减是事实，但 warehouse 可行性验证是第一关——如果基础设施本身跑不通，再多的另类数据也是空中楼阁。DeepSeek 指出传统因子 alpha 已衰减是对的，但时机判断有误。应该先验证管道，再按 P1.5 → P1 → P0.5 的顺序依次引入北向/融资融券/限售解禁等另类因子，而不是一开始就上所有东西造成调试复杂度爆炸。

**支持/反对谁：** 阶段性支持 Main，长期蓝图吸收 DeepSeek。

**补充论据：** A股市场传统因子（PE、PB、动量等）的 IC 确实在 2015-2020 年后系统性衰减，但这些因子的衰减本身就是一个 alpha 信号——说明市场散户结构在变。另类因子（北向持仓变动、融资融券余额、限售解禁比例）的有效性恰恰建立在散户占比高、定价效率低的基础上。建议明确两阶段：① P1 用传统因子验证管道；② P1.5 开始引入北向因子作为首个另类数据，并设立独立 IC 追踪。

## 分歧2：筹码增强优先级

**我的立场：支持 Main 保持 P3，但将筹码数据入仓设为 P2 触发条件之一。**

筹码数据的 alpha 潜力我完全认同——A股散户占比高，主力筹码分布对价格有显著预测能力。但"alpha 衰减慢"和"数据已就绪"是两回事。DeepSeek 的 P3→P1.5 升级建议在逻辑上没问题，但如果筹码数据还没入仓，强行提升优先级只会造成 P1.5 里程碑悬空。

**支持/反对谁：** 支持 Main 的数据现实主义，但要求设置硬性触发条件。

**补充论据：** 建议将"筹码数据入仓"从 P3 的 optional 升级为 P2 的必须完成项。一旦warehouse验证了传统因子的管道，立即启动筹码数据接入，并将其作为 P1.5 的主推 alpha 来源，而不是让它在 P3 慢慢排队。这样 DeepSeek 的核心诉求（筹码 alpha 不应被忽视）得到了优先级保障，同时不打破现有开发节奏。

## 分歧3：qant失败诊断

**我的立场：值得花时间诊断，但作为平行任务而非阻塞项。**

Main 说 qant 方法论有误（naive random），这点我部分同意——qant 的随机采样方法论确实存在理论缺陷，但-8.78% 的结果本身是有信息的。即使方法论有问题，知道"哪些因子类别拖后腿"能帮助我们理解 canonical 数据的相对优劣。关键是不必等诊断完再推进，可以并行跑 canonical 重跑，同时诊断。

**支持/反对谁：** 部分支持 DeepSeek，诊断有价值；但反对让诊断阻塞主流程。

**补充论据：** 诊断的真正价值不在于解释 qant 为什么失败，而在于建立一套因子类别 IC 基线——比如发现"情绪类因子在 qant 的 random sampling 下最差"，这能帮助我们预判 canonical 也会面临类似问题。建议：诊断只需2-3小时，不应成为阻塞项；如果诊断发现 canonical 的优势因子类别恰好是 qant 的弱势因子，那-8.78% 反而是一个反向验证。

## 分歧4：S1门槛严格度

**我的立场：维持 IC t-stat>1.5 或 bootstrap p<0.10 双轨并行，拒绝回到原门槛。**

这是我自己 original review 的核心论点，我坚持不动。IC t-stat>2 + bootstrap p<0.05 同时满足的概率在金融数据噪声环境下极低（约5%×5%=0.25%的通过率），这意味着绝大多数真实 alpha 因子会在 S1 就被错误拒绝。更严重的是，这个门槛会造成严重的 selection bias——只有高波动区的因子能通过，而低波动区的稳健 alpha 因子会被系统性遗漏。

**支持/反对谁：** 支持 Main 增加"弱 alpha 候选"路径；反对 DeepSeek 暗示维持原门槛的立场。

**补充论据：** "弱 alpha 候选"路径和我的双轨方案本质相同——都是放宽 S1 筛选条件，保留更多候选进入后续验证。区别在于实现方式：Main 倾向于加一个特殊通道，我倾向于改双轨条件本身。我愿意接受 Main 的"弱 alpha 候选"实现方式，只要明确该通道的通过率上限（不超过主通道的30%），避免把 S1 变成走过场。

## 分歧5：容量测试方法

**我的立场：S1 做简单但有效的过滤，拒绝过度复杂化；按市值分档纳入 S2 设计文档。**

Main 的判断是对的：S1 是粗筛阶段，不需要也没必要做市值分档的精细容量分析。但 S1 的简单过滤也不能只是"成交额>1亿"——这个阈值对大盘股合适，对小盘股却过于宽松，容易产生虚假 alpha。S1 容量测试至少应做到：①按成交额绝对值过滤（分市值档位设定不同阈值）；②记录各档位的因子 IC 分布，不做硬性拒绝但留下数据痕迹。

**支持/反对谁：** 阶段性支持 Main，但要求 S1 至少包含市值分层过滤而非单一阈值。

**补充论据：** DeepSeek 的市值分档建议是正确的方向，但属于 S2 的精细化工作而非 S1 的入门条件。建议在 S1 里程碑文档中明确写入："S2 将引入按市值分档的容量约束"，作为已知技术债，而不是在 S1 阶段就做过度工程化。容量测试前移到 P1.5（DeepSeek 的诉求）是对的，但前移的内容是"记录并分析"，而非"施加硬性拒绝"。

## 分歧6：embargo参数

**我的立场：建议10日，基于信息_decay理论和A股交易成本估算。**

三方都认为5日太紧，Main 和 DeepSeek 都建议 10-15日，我建议取下限10日。embargo 的核心目的是防止未来函数（look-ahead bias）和短期价格操纵，10日基本覆盖了A股大多数短期信息不对称消散的时间窗口。调高到15日的边际收益有限（额外5天只能进一步降低极短期操纵的可能，但也会不成比例地损失有效数据点）。

**支持/反对谁：** 支持 Main 和 DeepSeek 的方向，具体值取中间偏保守。

**补充论据：** 量化领域对 embargo 的实证研究（如 Cont et al.）表明，超过10-15日的 embargo 对因子 IC 的边际改善趋于平缓。10日是一个有理论支撑的合理值，既能与学术界做法对齐（通常用5-10日），也能给中国A股的特殊市场结构留出余量（散户交易集中、T+1制度）。建议不要设置固定 embargo 而是采用 IC 衰减曲线决定——当因子 IC 在 embargo>7日后基本稳定，就以该点为准，这样可以自适应不同因子的特性。

## 我愿意让步的点

1. **因子库策略**：接受 Main 的"先传统后另类"路线图，但要求在 P1 完成后、P1.5 启动前，正式评估并引入北向/融资融券因子
2. **筹码增强**：接受 Main 的 P3 优先级，但将筹码数据入仓升级为 P2 必须完成项而非 optional
3. **容量测试**：接受 Main 的 S1 简单过滤方案，但要求过滤条件包含市值分层而非单一成交额阈值
4. **embargo**：接受 Main/DeepSeek 的10-15日区间，取中间值10日

## 我坚持不动的点

1. **S1 门槛双轨放宽**：IC t-stat>1.5 或 bootstrap p<0.10，不接受回到 t-stat>2 + p<0.05 的原门槛；也不接受"原门槛+弱alpha通道"的双轨（这会让弱alpha通道变成主要通道）
2. **OOT 步数**：坚持12步不够，必须扩展到20-24步以覆盖2年+的样本外验证，这是尾部风险管理的最低要求
3. **尾部风险指标**：除了 IC 和 Sharpe，必须在 S1 报告模板中加入 Max Drawdown、VaR(5%)、Sortino Ratio 等尾部指标，这是机构投资者最关心的风控维度
4. **深度模型降级**：Arc B390 跳票概率>80%，S1/S2 阶段必须基于 LightGBM/XGBoost，不预设 Transformer/深度因子挖掘路线，除非 Arc B390 实际可用
