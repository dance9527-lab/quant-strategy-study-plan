## 二、回应 Main 和 Coder：成本/工程复杂度评估

### 2.1 Main 的成本分析：核心数据我完全认同

Main 给出的关键数据：

> 日调仓换手率约为月调仓的21倍，年化成本约75.6%

让我独立验证这个数字：

```
月调仓：
  月均换手率 ~50%
  年化交易成本 = 50% × 12 × (0.015%×2 + 0.1%×0.5 + 0.1%) = 50% × 12 × 0.13% ≈ 7.8%

日调仓：
  日均换手率 ~30%（假设）
  年化交易成本 = 30% × 250 × 0.13% ≈ 9.75% × ... 等等
```

重新计算：
- 单边成本 = 佣金万1.5×2 + 印花税千1×0.5（仅卖出）+ 过户费 + 冲击
- 简化：单次调仓（一买一卖）成本约 0.15% - 0.25%（含冲击）
- 月调仓：年换手率 ~600%（50%×12），年化成本 ~600% × 0.13% ≈ 7.8%
- 日调仓：年换手率 ~7500%（30%×250），年化成本 ~7500% × 0.10% ≈ **75%**

**Main 的75.6%估算合理**。即使按乐观假设（更低的冲击成本），日调仓年化成本也在50%以上。

**关键推论**：日调仓策略需要额外产生50-75个百分点的年化alpha才能覆盖成本。在A股全市场截面因子策略中，这是**不可能的**。

日频因子的alpha信号极弱，叠加极高的交易成本，日调仓策略在实盘中几乎确定为负收益。

## 四、如果保留S1-D：具体设计建议

### 4.1 明确S1-D的定位

**建议在文档中显式声明**：

> S1-D（日选股）是**研究主线**，目标是验证短期因子的预测能力和alpha衰减曲线。S1-D的输出**不直接用于实盘交易**。S1-D的成本后表现仅作为参考，不作为策略keep/drop的依据。

这个声明的价值：
1. 避免将S1-D的学术发现误读为实盘可行性
2. 降低对S1-D的工程投入要求（不需要生产级订单状态机）
3. 明确S1-D与S1-M的主次关系

**不建议为S1-D构建完整的实盘级系统**。建议：

#### A. 简化订单模拟（而非真实订单状态机）

```python

                                  prices, tradability):
        """
        简化逻辑：
        1. 计算目标持仓 vs 当前持仓的差异
        2. 按比例缩放（考虑100股整数手）
        3. 检查涨跌停/停牌（可交易性）
        4. 不可交易的持仓保持不变
        5. 成交部分按close价成交（不模拟盘中冲击）
        6. 成本按固定比例扣减（千1.5双边）
        """
        pass
```

工期：3-5天（vs 完整状态机的8-12天）

#### B. 成本敏感性用固定参数，而非动态模型

```python
# 简化成本模型
COST_PER_TRADE = {
    'commission': 0.00015,  # 万1.5双边
    'stamp_tax': 0.0005,    # 千1（仅卖出）
    'impact': 0.001,        # 固定冲击假设
}
```

不使用L1日频冲击模型、参与率模型等复杂成本估计——这些对研究目的来说过于精细。

#### C. Walk-forward日历：日度决策，月度汇总报告

```
S1-D Walk-Forward 设计：
- 决策频率：每日盘后生成候选清单
- 重训频率：每63交易日（与S1-M一致）
- OOT报告：按交易日记录IC/收益，但汇总为月度/季度/年度报告
- 最低验收：24个月度汇总step（与S1-M对齐）
```

#### D. 核心输出：因子频率分析报告

S1-D的最终产出应该是：

| 输出 | 内容 | 用途 |
|---|---|---|
| 日频IC衰减曲线 | IC(1日), IC(5日), IC(10日), IC(20日) | 判断因子的最佳预测频率 |
| 成本后alpha衰减 | 扣除成本后1日/5日/20日策略的累计收益 | 判断短期alpha是否被成本吞噬 |
| 因子持续性分析 | 持仓1天 vs 5天 vs 20天的收益分布 | 为S1-M的调仓频率提供依据 |
| 信号衰减半衰期 | 因子信号从产生到失效的时间 | 决定S1-M的最优调仓周期 |
