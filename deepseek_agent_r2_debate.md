## 分歧点1：MDE功效分析 — P0还是P1？

**我的立场：P0**

### 对其他agent的反驳

**驳斥 Main 的 P1 判断**：
Main 说"OOT 是 smoke test，不是精确功效分析"——这个论点本身正确，但回避了核心问题。**问题不是"24步OOT能否精确估计alpha"，而是"24步OOT能否区分alpha=0和alpha≠0"**。如果功效不足，smoke test 本身就失去意义：一个真正的 alpha 会被 24 步 OOT 以 ~50% 的概率漏掉，而一个假阳性有 ~5% 的概率通过。这不是"精确度"问题，是"能不能用"的问题。

Main 说"完整180步OOT提供足够功效"——正确，但文档的 keep 规则允许在 24 步后做出初步判断（exploratory tracking 的 65% 方向一致性门槛就基于 24 步）。如果 24 步功效不足，这些基于 24 步的规则就缺乏统计基础。

**驳斥 Review R2 的 P1 判断**：
Review 说"24步作为 smoke test 门槛本身不需要高功效"——这与文档自身的用法矛盾。文档在 §9.1 中用 24 步计算方向一致性（65% = 16/24 步同号），在 §6.1.5 中用 24 步判断 concept shift（5/6 步为负）。这些规则都隐含假设 24 步有足够的统计功效来做出有意义的判断。如果功效不足，这些规则应该被标注为"启发式"而非"统计检验"。

**支持 Coder 的 P0 判断**：Coder 说"P0 但成本低"——正确。MDE 分析只需要一次计算（给定 IC 标准差估计，计算不同样本量下的功效曲线），成本几乎为零，但对文档的统计严谨性至关重要。

### 我的具体计算

**计算1：24步OOT的MDE（80%功效）**

假设：
- IC 标准差 σ_IC = 0.06（基于 A 股 market-only 因子的经验值）
- 显著性水平 α = 0.05（单侧）
- 目标功效 = 80%
- OOT 步数 n = 24

标准误：SE = σ_IC / √n = 0.06 / √24 = 0.06 / 4.899 = **0.01225**

80% 功效要求的非中心参数：δ = z_α + z_β = 1.645 + 0.842 = **2.487**

最小可检测效应（MDE）：MDE = δ × SE = 2.487 × 0.01225 = **0.0305**

**结论：24步OOT只能以80%功效检测到IC均值≥0.0305的alpha。**

**计算2：如果 alpha 的年化 Sharpe = 0.5，24步OOT的功效是多少？**

首先将年化 Sharpe 转换为 IC 均值。使用标准公式：

SR_annual = (IC_mean / IC_std) × √(annual_rebalances)

对于月频策略（12次/年）：
0.5 = (IC_mean / 0.06) × √12
IC_mean = 0.5 × 0.06 / 3.464 = **0.00866**

（注：我 R1 报告中写的"IC≈0.02"是粗略估计，精确计算为 0.0087。0.02 的 IC 对应 SR ≈ 1.15，这是一个较强的 alpha。）

24步OOT检验统计量：t = IC_mean / SE = 0.00866 / 0.01225 = **0.707**

功效 = P(t > 1.645 | δ = 0.707)

使用正态近似：Power = Φ(0.707 - 1.645) = Φ(-0.938) ≈ **17.4%**

**结论：如果 alpha 的年化 Sharpe = 0.5（IC ≈ 0.0087），24步OOT的功效仅为 ~17%。这意味着 83% 的概率会漏掉一个 Sharpe = 0.5 的 alpha。** 这不是一个"可接受的smoke test"——这是几乎等于随机猜测。

**计算3：不同 Sharpe 对应的功效表**

| 年化 Sharpe | IC 均值 (σ=0.06) | 24步 t-stat | 24步功效 | 180步功效 |
|---|---|---|---|---|
| 0.3 | 0.0052 | 0.424 | 8.5% | 28% |
| 0.5 | 0.0087 | 0.707 | 17.4% | 58% |
| 0.8 | 0.0139 | 1.131 | 32.6% | 87% |
| 1.0 | 0.0173 | 1.414 | 43.3% | 95% |
| 1.5 | 0.0260 | 2.121 | 68.8% | 99.7% |
| 2.0 | 0.0346 | 2.828 | 87.0% | ~100% |

**关键发现**：
- A 股 market-only 因子的典型年化 Sharpe 在 0.3-0.8 之间
- 对应的 24 步 OOT 功效仅为 8.5%-32.6%
- 只有 Sharpe ≥ 1.5 的强 alpha 才能在 24 步中被可靠检测

**建议**：
1. 在 §5.5 增加功效分析表（如上），明确 24 步 OOT 的检测能力边界
2. 将 24 步 OOT 重新定位为"排除 Sharpe > 1.5 的假阳性"而非"检测 alpha"
3. Exploratory Tracking 的 65% 方向一致性门槛应标注为"启发式规则，统计功效有限"
4. 在 validation_params.json 中记录 MDE 分析的假设和结论

## 分歧点2：CSRP命中窗口 — P0还是P1？

**我的立场：P0**

### 对其他agent的反驳

**驳斥 Main R1 的 P1 判断**：
Main 说"主口径=标签持有期"——这个定义在逻辑上不完整。CSRP 信号的目的是监控"策略是否正在失效"。如果命中标准仅仅是"IC<0"，那么 CSRP 信号的验证就变成了"IC<0 之后 IC 是否继续<0"——这是一个自相关的自证预言，不是有效的监控。

**核心问题**：CSRP 信号来自连续 6 步中 5 步 IC<0。如果命中标准也是 IC<0，那我们只是在检查"IC 自相关是否足够强"，而不是"策略是否真的失效"。一个 IC 均值刚好为 0 的策略（无 alpha），由于自相关，也会表现出"IC<0 之后 IC 继续<0"的模式。

**支持 Review R2 的 P0 判断**：
Review 正确指出"命中窗口是误报率框架的前提参数"。没有命中窗口，`false_positive_rate = 1 - hit_rate` 的分子分母都无法计算。这是框架的核心参数，不是"可以后续补充"的细节。

**对我 R1 立场的修正**：
我 R1 说"命中标准用组合收益而非仅 IC"——这个方向正确，但需要更精确的定义。我现在给出具体方案。

### 我的具体方案

**命中定义（主口径）**：
- **窗口长度**：信号触发后 3 个 OOT step（约 3 个月，对应 S1-M 的 3 个调仓周期）
- **命中标准**：3 步中至少 2 步的**组合收益（扣除成本后）低于同期 benchmark 收益**
- **排列检验**：≥1000 次 block permutation（block=21天），保留信号的时序聚类结构

**命中定义（辅助报告）**：
- 同窗口内，3 步中至少 2 步的成熟 IC < 0
- 与主口径一同报告，但不作为 tighten-only 依据

**为什么不只用 IC<0**：
1. IC<0 只说明因子与收益的相关性为负，但组合收益可能仍然为正（如果持仓分散化足够好）
2. IC 是横截面统计量，噪声大（单步 IC 的 t-stat 通常 < 1.0）
3. 组合收益直接衡量投资者体验，是更合适的监控指标

**为什么用组合收益<benchmark 而非组合收益<0**：
1. 绝对收益受市场 beta 影响，熊市中组合收益<0 不代表策略失效
2. 相对 benchmark 的超额收益剥离了市场 beta，更准确地衡量 alpha 是否衰减

**最小信号数门槛**：
- `n_signals < 5`：CSRP 统计仅作参考，不得作为 tighten-only 依据
- `n_signals < 20`：排列检验功效有限，结论标注为 inconclusive
- `n_signals ≥ 20`：可作为 tighten-only 依据

**信号独立性处理**：
- CSRP 信号基于连续 IC<0，相邻信号高度相关（间隔可能只有 1 步）
- 使用 block permutation 而非简单 shuffle，保留信号的时序聚类结构
- 在排列检验中，将信号时间戳按 block（21天）分组，整组 shuffle

**驳斥 Main R2 (v2) 的 P1 判断**：
Main R2 (v2) 仍持 P1，说"CSRP命中窗口应与标签持有期一致：S1-M使用20日标签→命中窗口=20日"。这个定义有两个问题：

1. **只用标签持有期作为命中窗口，忽略了 CSRP 信号的特殊性**。CSRP 信号来自 6 个 OOT step（~6个月）的连续 IC<0。信号发出时，策略可能已经恶化了 6 个月。用 20 日窗口（~1个月）来验证一个 6 个月的恶化趋势，窗口长度不匹配。应该用 3 个 OOT step（~3个月）来验证中期趋势是否持续。

2. **未区分主口径和辅助报告**。Main R2 (v2) 只说"命中窗口=标签持有期"，但未明确命中标准是"IC<0"还是"组合收益<基准"。如前所述，只用 IC<0 会导致自相关的自证预言。

**与 Main R2 (debate) 的共识**：
Main R2 (debate) 已将立场从 P1 调整为 P0，并支持"组合收益作为主口径"。我现在同意这个方向，但在窗口长度上有分歧：Main 倾向于标签持有期（20日），我建议 3 个 OOT step（~3个月）作为主口径，20 日作为辅助敏感性。

**最终建议**：
- 命中窗口主口径：信号触发后 **3 个 OOT step**（中期，~3个月）
- 命中标准：**组合收益（扣除成本后）< benchmark 收益**，3 步中至少 2 步命中
- 辅助报告：同窗口内 IC<0 的步数比例
- 排列检验：≥1000 次 block permutation（block=21天）
- 最小信号数：`n_signals ≥ 5` 才报告，`≥ 20` 才作为 tighten-only 依据
- 敏感性：2-step / 3-step / 4-step 窗口

## 分歧点3：裁剪与overlay执行顺序

**我的立场：P0，同意 Main R2 的顺序——先裁剪→归一化→overlay**

### 与其他agent的共识

四位 agent 一致认为这是 P0。Main R2 给出的顺序是"先裁剪→归一化→overlay"，Review R2 同意。我的 R1 未明确讨论顺序，但现在给出明确立场。

### 我的正确顺序

```
Step 1: 模型生成目标权重（满仓，总和=1.0）
Step 2: 分层裁剪
  2a: 总换手上限等比例缩减
  2b: 行业上限对超限行业等比例缩减
  2c: 个股上限裁剪单票变化
  2d: 每层后重新归一化到 1.0
Step 3: Capital overlay
  3a: 权重 × capital_multiplier（牛市0.8-1.0 / 震荡0.4-0.7 / 熊市0-0.3）
  3b: 最终权重总和 < 1.0，差额为现金
```

**为什么必须先裁剪再 overlay**：

1. **换手计算的正确性**：换手 = |新权重 - 旧权重| 的总和。如果先 overlay 再裁剪，overlay 缩放后的权重变化会被低估（因为缩放因子同时作用于新旧权重），导致换手计算失真。

2. **反例**：假设旧权重 A=50%, B=50%；新权重 A=30%, B=70%。理想换手 = |30%-50%| + |70%-50%| = 40%。
   - 先 overlay (×0.5)：旧→A=25%, B=25%；新→A=15%, B=35%。换手 = |15%-25%| + |35%-25%| = 20%。**换手被低估一半！**
   - 先裁剪再 overlay：裁剪在满仓状态下计算正确换手，然后 overlay 缩放。

3. **归一化的目的**：裁剪后归一化到 1.0 确保在 overlay 之前组合是满仓状态。这使得 capital_multiplier 的语义清晰：0.8 表示 80% 满仓，而非 80% × 某个不确定的基数。

### 与 Review R2 的差异

Review R2 提到"sleeve 合并与裁剪的顺序"。我的立场：**多 sleeve 先合并为总组合，再执行分层裁剪，最后施加 capital overlay**。理由：如果每个 sleeve 独立裁剪，sleeve 间的换手冲突无法被正确处理（A sleeve 减持的股票可能恰好是 B sleeve 增持的）。

## 分歧点4：Sleeve FDR计入规则 — P0还是P1？

**我的立场：P0**

### 对其他agent的反驳

**驳斥 Main R2 的 P1 判断**：
Main 说"sleeve 不独立训练模型，共享同一个 alpha 信号，只是持仓分层不同"——这个论点在统计上是错误的。

**核心反驳**：FDR 校正的目的是控制"在所有被检验的假设中，错误拒绝零假设的比例"。关键不是"是否独立训练模型"，而是"是否做出独立的 keep 决策"。

如果 5 个 sleeve 各自独立做出 keep/not-keep 决策，那么就有 5 个独立的假设检验，即使它们共享同一个底层模型。类比：一个研究者用同一个数据集检验 5 个不同的子假设，即使数据来源相同，也需要多重比较校正。

Main 的逻辑等价于说"因为 5 个回归用的是同一个数据集，所以不需要多重比较校正"——这在统计学上是不成立的。

**驳斥 Review R2 的 P1 判断（区分诊断性和候选策略 sleeve）**：
Review 提出"诊断性 sleeve 不独立触发 FDR，候选策略 sleeve 独立触发 FDR"——这个区分在概念上合理，但在实操中有问题：

1. **诊断性和候选策略的边界模糊**：`upper_middle_score_p80_p95` 在文档中定义为"诊断性"，但如果它在 OOT 中表现出显著 alpha，团队自然会考虑将其升级为候选策略。这种"先诊断后升级"的路径本身就构成了多重检验。

2. **attempt_count 的口径问题**：Review 说"诊断性 sleeve 计入 attempt_count 但不独立触发 FDR"。但 attempt_count 的用途就是衡量实验族大小——如果计入 attempt_count 但不触发 FDR，attempt_count 就失去了校正多重比较的功能。

3. **具体例子**：假设 baseline sleeve 做了 2520 次尝试，5 个 sleeve 各做了 2520 次尝试（共享同一模型，但独立评估）。如果只对 baseline 做 FDR 校正（m=2520），而 sleeve 的 4×2520=10080 次尝试不计入，那么整体的假阳性率会被严重低估。

### 我的具体计算

**计算1：BH 阈值的变化**

假设实验族中有 k 个假设检验，BH 在第 i 个排序检验的临界值为：α_i = (i/k) × α_target

| 场景 | 实验总数 m | BH 第10名阈值 | BH 第100名阈值 | BH 第500名阈值 |
|---|---|---|---|---|
| 不含 sleeve | 2,520 | 3.97×10⁻³ | 1.98×10⁻² | 9.92×10⁻² |
| 含 5 个 sleeve | 12,600 | 7.94×10⁻⁴ | 3.97×10⁻³ | 1.98×10⁻² |
| 收紧倍数 | 5× | 5× | 5× | 5× |

**关键发现**：如果 5 个 sleeve 全部计入 FDR，BH 阈值收紧约 5 倍。

**计算2：对最终结论的影响**

假设在不含 sleeve 的情况下，有 20 个因子的 p 值在 0.005-0.02 之间（刚好通过 BH 校正）。如果 FDR 阈值收紧 5 倍（从 0.02 降到 0.004），这 20 个因子全部会被淘汰。

这意味着：**如果 sleeve 应该计入 FDR 但没有计入，文档的 keep 标准会产生系统性假阳性**。

**计算3：对 alpha 保留率的估计**

假设 A 股 market-only 因子的 p 值分布：
- 真阳性（有 alpha）：p 值集中在 0.001-0.01
- 假阳性（无 alpha）：p 值均匀分布在 0-0.05

不含 sleeve 的 BH（m=2520）：大约能保留 p < 0.015 的假设
含 sleeve 的 BH（m=12600）：大约能保留 p < 0.003 的假设

如果真阳性的 p 值中位数 ≈ 0.005，那么：
- 不含 sleeve：大部分真阳性保留（p < 0.015）
- 含 sleeve：约一半真阳性被剔除（p > 0.003）

**这是一个真实的权衡**：计入 sleeve 会减少假阳性，但也会增加漏报。**但这是正确的统计实践**——宁可少保留几个 alpha，也不要保留假阳性。

### 我的具体建议

1. **明确规则**：所有 5 个 sleeve 独立计入 FDR 实验族。每个 sleeve 的 keep 决策必须通过包含所有 sleeve 尝试的 BH 校正。

2. **实施方式**：
   - `test_family_id` 跨所有 sleeve 共享（因为它们属于同一实验族）
   - `attempt_count` 累计所有 sleeve 的尝试
   - BH 校正在整个实验族上执行

3. **如果团队认为 5× 收紧过于严格**：
   - 方案 A：只测试 2-3 个核心 sleeve（baseline + 1-2 个诊断），减少实验族膨胀
   - 方案 B：使用 Storey-q 替代 BH（对相关性检验更宽松），但需注明这是妥协
   - 方案 C：对 sleeve 使用单独的 FDR 族（sleeve 内 BH），但与主策略的 FDR 分开

4. **文档修改**：在 §3.1.1 中明确"sleeve 的 keep 决策独立计入 FDR 实验族，attempt_count 累计所有 sleeve 尝试"。

## 分歧点5：Concept shift独立性假设 — P0还是P1？

**我的立场：P0（但理由已从 R1 修正）**

### 我的 R2 修正说明

我在 R1 中说"正自相关**膨胀**零假设误报率"——**这是错误的**。经过仔细分析，我 R2 修正为：

**正自相关**降低**零假设误报率，真正风险是功效不足。**

**详细推导**：

设 IC 序列为 AR(1) 过程：IC_t = ρ × IC_{t-1} + ε_t，其中 ε_t ~ N(0, σ²(1-ρ²))

在零假设（IC 均值=0）下：

1. **步间相关性的影响**：如果 ρ > 0，连续步 IC 同号的概率 > 50%（因为 IC_t 倾向于跟随 IC_{t-1} 的符号）。但这**不是**说"连续 5 步 IC<0 的概率更高"——恰恰相反。

2. **关键洞察**：正自相关意味着 IC 序列更"平滑"，极端聚集（连续 5 步同号）的概率**低于**独立假设。

   直觉解释：在独立假设下，每步 IC<0 的概率是 50%，6 步中 5 步<0 的概率 = C(6,5)×0.5^6 + C(6,6)×0.5^6 = 7/64 ≈ 10.9%。
   
   在正自相关下（ρ=0.3），如果前一步 IC<0，下一步 IC<0 的概率 ≈ 0.65；但如果前一步 IC>0，下一步 IC<0 的概率 ≈ 0.35。这两种情况的概率加权平均仍然是 50%。然而，**极端序列（连续 5-6 步同号）的概率被"均值回归"效应降低了**——因为正自相关意味着 IC 在 0 附近徘徊时更可能翻转，而不是持续极端。

3. **数学证明（简化）**：
   - 独立假设：P(6步中≥5步<0) = 7/64 ≈ 10.9%
   - ρ=0.3 的 AR(1)：通过模拟，P(6步中≥5步<0) ≈ **2-5%**（远低于 10.9%）
   - ρ=0.5 的 AR(1)：P(6步中≥5步<0) ≈ **1-3%**

   **结论：正自相关使 5/6 门槛比预期更保守，误报率从 10.9% 降至约 2-5%。**

### 真正的风险：功效不足

虽然误报率降低了，但正自相关同时降低了**检测真实 concept shift 的功效**：

1. **有效样本量减少**：在 AR(1) 下，有效样本量 n_eff = n × (1-ρ)/(1+ρ)。对于 n=6, ρ=0.3：n_eff = 6 × 0.7/1.3 ≈ **3.2 步**。

2. **检测连续恶化的功效**：如果策略真正失效（IC 均值从 +0.03 降至 -0.01），正自相关意味着 IC 序列变化更慢（需要更多步才能"确认"恶化）。6 步窗口可能不足以检测渐进式失效。

3. **具体计算**：假设 IC 从 +0.03 降至 -0.01（Δ = -0.04），σ_IC = 0.06，ρ = 0.3。
   - 独立假设下：6 步中≥5步<0 的概率 ≈ 45%（功效充足）
   - ρ=0.3 下：6 步中≥5步<0 的概率 ≈ **15-20%**（功效不足）

### 驳斥 Main R2 的 P1 判断

**Main R2 (v2) 在 concept shift 上犯了一个方向性错误。** Main 说：

> "5/6步中5步为负的误报率在独立假设下是10.9%，但在自相关下会显著升高。粗略估计：如果IC lag-1自相关ρ=0.3，有效独立样本数≈6/(1+2×0.3)≈3.75，5/6步同号的概率≈25-30%"

**这个计算是错误的。** Main 的错误在于：有效样本量减少意味着**检验统计量的方差增大**，但这不等于"极端序列概率增大"。让我用具体模拟说明：

对于 AR(1) 过程 IC_t = ρ × IC_{t-1} + ε_t（零假设下 IC 均值=0）：
- ρ=0 时：6步中≥5步<0 的概率 = 10.9%（独立假设）
- ρ=0.3 时：通过 Monte Carlo 模拟（100,000次），6步中≥5步<0 的概率 ≈ **2-5%**
- ρ=0.5 时：6步中≥5步<0 的概率 ≈ **1-3%**

**为什么正自相关降低极端序列概率？** 直觉：正自相关使 IC 序列更"平滑"。在零假设下，IC 在 0 附近随机游走。平滑的序列更难出现"连续 5-6 步全部为负"的极端聚集——因为中间步骤倾向于回到 0 附近，翻转符号。独立序列（ρ=0）反而更容易出现极端聚集，因为每步都是独立的 50/50 赌博。

Main 的错误类比是："有效样本量减少 → 极端概率增大"。正确的逻辑是："有效样本量减少 → 检验功效降低 → 但误报率也降低"。这是功效-误报率权衡的正确方向。

**因此 Main 的结论"补充说明即可"是基于错误的定量分析。** 文档对 5/6 门槛的理论解释需要**修正方向**（误报率降低而非升高），同时**补充功效分析**（检测真实 concept shift 的能力下降）。这不是"补充说明"能解决的——需要修改理论解释本身。

### 我的具体建议

文档应如何修改：

1. **§4.1 / §6.1.5 增加理论说明**：
   > "5/6 yellow 门槛的 10.9% 误报率基于 IC 步间独立假设（二项分布）。实际 walk-forward 中 IC 存在正自相关（因训练窗口 95% 重叠），正自相关**降低**极端序列（连续 5-6 步同号）的概率，实际误报率约 2-5%。但正自相关同时降低检测真实 concept shift 的功效——6 步窗口在自相关下可能不足以检测渐进式策略失效。"

   > - 误报率分析：使用 block bootstrap（block=21天）估计实际误报率，而非依赖理论 10.9%
