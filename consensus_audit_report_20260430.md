## 二、共识达成的改进项

### 🔴 P0 — 必须立即修复（3项）

#### 1. S1通过门槛放宽
**现状**：IC t-stat>2 + bootstrap p<0.05 + 12 OOT steps + holdout不失效 — 四项同时满足概率极低
**共识方案**：
- 增加"弱alpha候选"路径：IC方向一致 + 分年度正率>70% + 成本后不为负 → 标记为candidate（非discard）
- 主路径门槛调整：IC t-stat>1.5（原>2）或bootstrap p<0.10（原<0.05），满足其一即可
- 增加多重检验校正：候选因子>20个时，必须报告FDR校正后的显著性

#### 2. ST PIT源缺口处理
**现状**：沪/北历史ST缺PIT源，只有深市历史较完整
**共识方案**：
- 方案A（优先）：补充历史ST公告数据（从AkShare或交易所公告）
- 方案B（保底）：将ST相关因子降级为"非PIT合规，仅作参考信号"，不进入keep决策
- 在S1报告中明确标注ST过滤对因子IC的影响（有/无ST过滤对比）

#### 3. 容量基线测试前移
**现状**：容量测试在P3阶段，太晚
**共识方案**：
- S1阶段增加最简容量过滤：日均成交额>1000万
- S1报告中包含基础容量指标：持仓市值/日均成交额比值
- 精细容量分析（分档滑点/冲击/参与率）保留在S2

### 🟠 P1 — 重要改进（6项）

#### 4. embargo参数调整
**现状**：embargo 5个交易日
**共识方案**：调整为10-15日
- 理由：A股动量因子有3-5日持续效应，5日embargo不足以消除自相关泄漏
- 建议值：embargo_days = max(10, ceil(调仓周期/3))

#### 5. OOT steps扩展
**现状**：最少12步OOT，覆盖约1年
**共识方案**：
- 方案A：缩短训练窗口至3年，保持21日调仓 → 约24步OOT，覆盖2年
- 方案B：保持5年训练，但OOT覆盖2005-2026全期（约50步），按子期间报告
- 无论哪种方案，必须包含分年度和分市场状态的子期间分析

#### 6. 因子正交化流程
**现状**：缺失
**共识方案**：在ICIR加权和Ridge之间增加Gram-Schmidt正交化步骤
- 目的：防止因子共线性导致正则化失效
- 产物：正交化前后因子相关性矩阵对比

#### 7. 北向资金+融资融券数据纳入P1
**现状**：放在P3
**共识方案**：作为P1的并行数据源，与传统因子同步测试
- AkShare已可提供北向资金和融资融券数据
- 不阻塞传统因子基线，但应在S1阶段同步产出IC报告
- PIT审计可延后，先标记为"candidate"

#### 8. regime断裂检测模块
**现状**：缺失
**共识方案**：
- 在walk-forward的每个OOT step中单独报告regime断裂期表现
- 增加简单regime检测：基于波动率聚类或连续IC为负的检测
- 风险开关中加入"regime断裂保护"：连续5日因子IC为负时自动减仓信号

#### 9. 尾部风险指标
**现状**：缺失
**共识方案**：在S1验证中增加VaR 95%、CVaR 99%、最大回撤持续期

### 🟡 P2 — 增强稳健性（5项）

#### 10. 筹码数据ETL提前
**现状**：P3
**共识方案**：P1阶段启动ETL，验证通过后升级到P1.5

#### 11. purge窗口扩展
**共识方案**：purge >= max(horizon*3, 40)（原max(horizon,20)）

#### 12. 回测版本管理
**共识方案**：每次OOT测试记录git commit hash + 数据版本 + validation_params.json SHA256

#### 13. 动态交易成本模型
**共识方案**：区分高/低流动性两套成本参数

#### 14. 季节性效应处理
**共识方案**：在因子中加入月份哑变量，或确保walk-forward训练窗口包含完整季节性周期

### 🟢 P3 — 锦上添花（3项）

#### 15. 深度模型降级
**共识方案**：放弃PatchTST/TFT，改用轻量LSTM或1D-CNN，或定位为"云端实验"

#### 16. 涨跌停板磁吸效应模拟
**共识方案**：在回测引擎中加入磁吸模拟

#### 17. 因子解释性评分
**共识方案**：在因子评级中增加"解释性评分"维度

## 三、分歧裁决记录

### 分歧1：因子库"教科书化"

| 立场 | 内容 |
|------|------|
| DeepSeek | 传统因子alpha已衰减，应一开始就纳入另类数据因子 |
| Main | 先用传统因子验证warehouse可行性，再扩展 |
| **裁决** | **折中方案**：S1同时包含传统因子+3-5个另类数据因子（北向/融资融券），但另类数据因子PIT审计可延后，先标记candidate。理由：DeepSeek对alpha衰减的判断有道理，但不应阻塞基线验证 |

### 分歧2：筹码增强优先级

| 立场 | 内容 |
|------|------|
| DeepSeek | P3 → P1.5（散户占比高，alpha衰减慢） |
| Main | 保持P3（数据未入仓） |
| **裁决** | **折中方案**：P1阶段启动筹码数据ETL，验证通过后升级到P1.5。理由：DeepSeek对alpha潜力的判断有道理，但数据就绪度是硬约束 |

### 分歧3：OOT steps覆盖期

| 立场 | 内容 |
|------|------|
| Review | 12步只覆盖1年，需20-24步 |
| Main | 12步足够 |
| **裁决** | **Review胜**：12步确实只覆盖1年，无法观测因子衰减全周期。建议3年训练窗口/21日调仓→约24步OOT |

### 分歧4：qant失败诊断

| 立场 | 内容 |
|------|------|
| DeepSeek | 应先诊断qant哪些因子拖后腿 |
| Main | qant方法论有误，不值得诊断 |
| **裁决** | **Main胜**：qant的random 8/2是方法论错误（label泄漏+验证污染），诊断其因子无意义。正确做法是用warehouse canonical数据重新walk-forward |

### 分歧5：深度模型可行性

| 立场 | 内容 |
|------|------|
| Review | P4跳票概率>80%，应降级 |
| DeepSeek | 信噪比低，深度模型优势不明确 |
| Main | 可行性是硬约束 |
| **裁决** | **三方共识**：深度模型降级为轻量方案（LSTM/1D-CNN），或定位为云端实验 |

## 四、改进后的验证框架参数

| 参数 | 原值 | 改进值 | 改进理由 |
|------|------|--------|---------|
| 训练窗口 | 5年 | 3年（默认）或5年（可选） | 3年可产出更多OOT steps |
| 调仓窗口 | 21交易日 | 21交易日（保持） | — |
| purge | >=max(horizon,20) | >=max(horizon*3,40) | 更保守的样本隔离 |
| embargo | 5日 | 10-15日 | A股因子自相关持续期更长 |
| 最少OOT steps | 12 | 24（3年训练）或12+分年度（5年训练） | 覆盖更完整的因子衰减周期 |
| S1 IC t-stat | >2 | >1.5 或 bootstrap p<0.10 | 避免淘汰弱但稳定的alpha |
| S1 holdout | 不失效 | 方向一致+Sharpe>0 | 放宽以容纳弱alpha |
| 多重检验 | 无 | FDR校正（因子>20个时） | 防止数据挖掘偏差 |

## 九、三方保留意见

### Main Agent 保留
- qant实验不值得花时间诊断，应直接用canonical数据重跑
- 风险开关的100/60/30/0数字需要先做市场状态研究才能确定

### Review Agent 保留
- 深度模型P4跳票概率>80%，应明确降级或放弃
- PIT合规性不能只看warehouse层，实验层需要独立审计
- 尾部风险指标（VaR/CVaR）应进入S1门槛

### DeepSeek Agent 保留
- 传统因子alpha已衰减，应更激进地转向另类数据
- 容量需要按市值分档分析，1亿是乐观估计
- 筹码增强应提升到P1.5

## 十、最终结论

该计划的**数据底座工程**（warehouse 6轮审计+P1前置）和**验证框架设计**（walk-forward+purge+embargo）是其最大优势，在A股量化研究计划中属于少见的严谨水平。

**核心短板**：
1. S1门槛过严会淘汰有效弱alpha → 已放宽
2. 因子库过于传统 → 已增加另类数据因子
3. 深度模型硬件不可行 → 已降级
4. 容量测试时机反了 → 已前移
5. 缺少时间线和里程碑 → 已补充

**共识评分**：**7.0/10**（改进后预计可达到8.0/10）
