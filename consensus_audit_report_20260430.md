## 三、最终共识方案

### 验证框架参数

| 参数 | 原值 | 最终值 | 依据 |
|------|------|--------|------|
| 训练窗口 | 5年 | **5年** | Review坚持，样本量充足 |
| 调仓窗口 | 21交易日 | **21交易日** | 保持 |
| purge | max(horizon,20) | **max(horizon*3,40)** | 更保守的样本隔离 |
| embargo | 5日 | **10日** | A股因子自相关实证（DeepSeek数据） |
| 最少OOT steps | 12 | **24步（每步21日）** | 覆盖2年，40%训练/测试比 |
| 分年度分析 | 无 | **必须** | 识别因子在不同年份/市场状态的表现 |
| S1 IC t-stat | >2 | **≥1.65 或 bootstrap p<0.10** | 折中于1.5和2.0，避免0.25%通过率 |
| S1 holdout | 不失效 | **方向一致+Sharpe>0** | 放宽以容纳弱alpha |
| bootstrap方法 | 无 | **Block Bootstrap, block=21日, ≥5000次** | 保持IC时间序列自相关结构 |
| 多重检验 | 无 | **FDR校正（因子>20个时）** | 防数据挖掘偏差 |
| 尾部风险 | 无 | **S1报告模板（记录不门槛）** | S2引入MaxDD/VaR/Sortino门槛 |

### Exploratory Tracking 机制

| 维度 | 最终方案 |
|------|---------|
| 命名 | **Exploratory Tracking** |
| 冷却期 | **≥6个月** |
| 方向一致性门槛 | **≥65%** |
| 入组合 | **否**（直到通过S1正式门槛） |
| 记录 | **完整记录含回顾性结论** |
| 用途 | 自我约束机制：记录接近但未达标的因子，防止重复测试同一弱信号，保持研究纪律 |

### 因子库扩展

- P1阶段：传统因子 + 3-5个另类数据因子（北向资金/融资融券/限售解禁）
- 筹码数据ETL：P1并行开发，P2必须完成
- 另类数据因子PIT审计可延后，先标记candidate

### 策略优先级

| 优先级 | 内容 | 变更 |
|--------|------|------|
| P0 | 实验层PIT/label/validation audit | 不变 |
| P1 | 多因子基线+北向/融资 | 新增另类数据因子 |
| P1.5 | 风险开关+筹码ETL | 筹码ETL提前 |
| P2 | 行业轮动+分红ETL+qant诊断(2天) | qant并行诊断 |
| P3 | 涨停事件 | — |
| P4 | 期权/轻量深度模型 | 深度模型降级 |

### 工程改进

- 深度模型：放弃PatchTST/TFT，改用轻量LSTM/1D-CNN
- 容量测试：S1分层过滤（按市值分档记录IC），S2精细分析
- 因子正交化：ICIR加权前增加Gram-Schmidt正交化
- 回测版本管理：每次OOT记录git commit + data version + params hash
