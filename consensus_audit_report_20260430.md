## 三、最终改进方案

### 验证框架参数

| 参数 | 原值 | 最终值 | 改进理由 |
|------|------|--------|---------|
| 训练窗口 | 5年 | 5年（保持） | Review坚持5年有道理，3年样本不足 |
| 调仓窗口 | 21交易日 | 21交易日 | 保持 |
| purge | max(horizon,20) | max(horizon*3,40) | 更保守的样本隔离 |
| embargo | 5日 | 10日 | A股因子自相关持续期（DeepSeek实证数据） |
| 最少OOT steps | 12 | 12 + 分年度分析 | 2:1裁决，分年度提供等效市场状态覆盖 |
| S1 IC t-stat | >2 | >1.65 或 p<0.10 | 折中于1.5和2.0，避免0.25%通过率 |
| S1 holdout | 不失效 | 方向一致+Sharpe>0 | 放宽以容纳弱alpha |
| 多重检验 | 无 | FDR校正（因子>20个时） | 防数据挖掘偏差 |
| 尾部风险 | 无 | S1报告模板（记录不门槛） | S2引入MaxDD/VaR/Sortino门槛 |
| 弱alpha | 无 | 探索性跟踪（不决策） | 记录方向性证据供后续研究 |

### 因子库扩展

- P1阶段：传统因子（估值/市值/流动性/动量/反转/波动率）+ 3-5个另类数据因子（北向资金/融资融券/限售解禁）
- 另类数据因子PIT审计可延后，先标记candidate
- 筹码数据ETL：P1阶段并行开发，P2必须完成

### 策略优先级调整

| 原优先级 | 调整后 | 变更 |
|---------|--------|------|
| P0 审计 | P0 审计 | 不变 |
| P1 多因子基线 | P1 多因子基线+北向/融资 | 新增另类数据因子 |
| P1.5 风险+分红 | P1.5 风险+筹码ETL | 筹码ETL提前 |
| P2 行业+qant | P2 行业+分红ETL+qant诊断 | qant诊断限2天 |
| P3 筹码/涨停 | P3 涨停事件 | 筹码已提前 |
| P4 期权/深度 | P4 期权/轻量深度 | 深度模型降级 |

### 工程改进

- 深度模型降级：放弃PatchTST/TFT，改用轻量LSTM/1D-CNN
- 容量测试：S1分层过滤（按市值分档记录IC），S2精细分析
- 回测版本管理：每次OOT记录git commit + data version + params hash
- 因子正交化：ICIR加权前增加Gram-Schmidt正交化

## 五、分歧裁决记录

### 裁决1：S1门槛
- DeepSeek提议t-stat>1.65 → **采纳**（折中于1.5和2.0）
- Review拒绝弱alpha候选通道 → **部分采纳**：保留但改名"探索性跟踪"
- Main的弱alpha候选路径 → **采纳**：改名+不进入决策流程

### 裁决2：OOT步数
- Review坚持20+步 → **未采纳**（2:1）
- Main+DeepSeek支持12步+分年度 → **采纳**
- 理由：5年训练窗口（Review坚持）+ 12步OOT + 分年度分析 = 等效市场状态覆盖

### 裁决3：尾部风险
- Review要求S1门槛 → **未采纳**
- Main要求S1报告+软门槛 → **部分采纳**：S1报告模板
- DeepSeek要求S2门槛 → **采纳**
- 最终：S1报告模板（记录不门槛），S2引入MaxDD/VaR/Sortino门槛

### 裁决4：embargo
- 三方共识10日 → **采纳**

### 裁决5：训练窗口
- Review坚持5年，Main原主张3年 → **采纳Review**：5年训练窗口
- 理由：A股数据量有限，3年窗口减少40%样本风险较大

## 八、结论

该计划的**数据底座工程**和**验证框架设计**是其最大优势。经过两轮讨论和裁决：

**核心改进**：
1. S1门槛放宽到t-stat>1.65或p<0.10（避免0.25%通过率）
2. embargo从5日调至10日（基于A股因子自相关实证）
3. 因子库增加另类数据因子（北向/融资融券）
4. 筹码数据ETL提前到P1并行
5. 深度模型降级为轻量方案
6. 容量测试前移到S1分层过滤
7. 尾部风险指标纳入S1报告

**共识评分**：**7.0/10**（改进后预计7.5/10）
